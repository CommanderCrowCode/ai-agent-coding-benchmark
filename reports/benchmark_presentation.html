Assessment Framework

Methodology

Two complementary rubrics provide both breadth and depth evaluation of each agent's output.

Relative Rubric (Weighted /10)

20 features scored 0–10 with weighted importance. Measures feature completeness, code quality, and adherence to specifications across the entire project surface area.

  • Weights sum to 100%, reflecting business priority
  • Inventory Router (15%) has the highest weight
  • Testing Suite (12%) and Database Schema (10%) follow
  • Weighted average produces the final relative score

Absolute Rubric (Points /200)

Eight categories scored on a fixed point scale. Tests whether the application actually runs, passes tests, and implements correct business logic — not just whether code exists.

  • "Does It Run?" (30 pts) — can the app actually start
  • "Test Suite" (35 pts) — test coverage and pass rate
  • "API Endpoints" (50 pts) — largest category
  • Security, Audit, Frontend round out assessment

Task Specification

All six agents received identical instructions to build a complete inventory management system with:

  • FastAPI backend with SQLAlchemy ORM
  • JWT authentication with role-based access
  • Full CRUD for products, suppliers, purchase orders
  • Payment processing and inventory tracking
  • Background tasks, email utilities, audit logging
  • Frontend dashboard, Docker deployment, testing
